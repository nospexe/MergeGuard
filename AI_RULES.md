# AI Usage Rules for MergeGuard

MergeGuard is an open-source FOSS hackathon project. We use AI tools
to accelerate development, but with clear rules to maintain code quality
and contributor understanding.

---

## Permitted AI Usage

- Using AI assistants (Claude, Copilot, etc.) to generate boilerplate
- Using AI to suggest implementations that contributors then review
- Using AI to write or improve documentation and docstrings
- Using AI to help debug failing tests

---

## Required Practices

**Understand what you commit.**
Every line of AI-generated code that enters this repository must be
understood by the contributor submitting it. If you cannot explain
what a piece of code does, do not commit it.

**Review before committing.**
AI-generated code must be reviewed for correctness, security, and
fit with the existing architecture before being committed.

**No blind copy-paste.**
Do not copy AI output directly into the codebase without reading and
understanding it. Adapt it to fit MergeGuard's conventions.

---

## Prohibited

- Committing code you do not understand
- Using AI to fabricate test results or benchmark numbers
- Using AI to generate fake commit history or changelogs
- Submitting AI-generated content as your own without disclosure

---

## Disclosure

If a significant portion of a PR was AI-assisted, note it in the PR
description. This is not penalised — it is encouraged as a practice
of transparency.
